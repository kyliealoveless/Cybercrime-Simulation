## User Manual

Every good documentation needs a user manual. By how I planned the code, this part is relatively easy. The way I planned my code, I already included the population size, cyber crimes being added, and how the ripple effect is spread through connections (indirectly), and the types of roles assigned. All this essentially means there is no need for user input.

To run my simulation to see different variations of randomized results in the .csv files "Final_Summary" and "Snapshots", you will simply make sure you can build the project, then run the task. The terminal will include a description of what this simulation is and will provide all my testing outputs so that you can see the process of how the outputs are created. If you wish to hide the testing outputs, I have commented throughout my code where the testing outputs are and you can simply comment them out and rebuild the project.

## Limitations

This is an undergrad research project so of course there will be limitations. Some of the limitations I have through this simulation is that the randomized values this simulation presents simplifies real-world behavior. Humans are very complicated beings and those complications should be simulated for more accurate results. Next, job loss was not completed. I had plans to add job loss to show further complications that come with cyber crime an worsens individual's mental state. I was too busy with implementing all the logistics for the ripple effect, recovery models, and summary files that I forgot to finish the logistics for job loss until I realized I did not have enough time with finals week. So I analyzed the results for what they had. This additionally makes my simulation less accurate than intended. Going more into the unpredictability of human lives, the recovery models are generalized. I took my interpretations from the external research I did of people healing from cyber crime and made estimates. Lastly, cyber crime is a heavily under reported crime due to the nature of cyber crime victim perceptions. This causes all my data to be slightly inaccurate due to the lack of data in general.
